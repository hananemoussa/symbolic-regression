```python
import numpy as np

def priority(item: float, bins: np.ndarray) -> np.ndarray:
    """
    Calculates priority scores for each bin based on the difference between the item size and the bin capacity.
    
    Args:
        item (float): Size of the item to be added to the bin.
        bins (np.ndarray): Array of capacities for each bin.
    
    Returns:
        np.ndarray: Array of same size as bins with priority scores for each bin.
    """
    # Calculate the penalty for each bin
    # The penalty is the negative difference between the item size and the bin capacity
    # This represents the "cost" of adding the item to each bin
    penalty = -np.abs(item - bins)
    
    # Calculate the priority score for each bin
    # The priority score is the reciprocal of the penalty, so the item with the smallest penalty gets the highest priority
    # This is based on the understanding that the item with the smallest penalty is the most valuable
    priority = 1 / penalty
    
    # Ensure that the priority scores are non-negative
    # This is because negative priority scores would imply that the item gets a negative reward
    priority = np.clip(priority, 0, None)
    
    return priority
```

Here's a brief explanation of the changes made to create this new function:

1.  **Inverse Penalty**: Instead of simply subtracting the bin capacity from the item size, we use the absolute difference (`np.abs`) to ensure that the penalty is always positive, regardless of whether the item is larger or smaller than the bin capacity.
2.  **Penalty Calculation**: We calculate the penalty by taking the absolute difference between the item size and the bin capacity. This ensures that the penalty is always positive, and we can calculate the priority score accordingly.
3.  **Priority Score**: The priority score is calculated as the reciprocal of the penalty. This represents the inverse of the penalty, where the item with the smallest penalty gets the highest priority.
4.  **Clipping**: We use the `np.clip` function to ensure that the priority scores are non-negative, which is necessary because negative priority scores would imply that the item gets a negative reward.

By applying these changes, the `priority()` function now calculates priority scores that are based on the inverse of the penalty, which provides a more accurate and meaningful representation of the item's value.
